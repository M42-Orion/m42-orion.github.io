  • 垃圾账户：ID 8-1000+（批量注册的机器人账户）
  • 命名规律：用户名长度均大于10字符，邮箱来自特定垃圾域名

二、攻击特征分析

特征 详情
邮箱模式 姓名+数字@子域名.主域名
典型样本 mackriley79@simu.elyquin.org、katharinaarchuleta91@seti.juxal.org
域名特点 算法生成的假域名，模拟.org组织域名增加可信度
用户名特点 英文姓名+数字组合，规避简单长度检测
攻击目的 疑似为后续垃圾邮件、仓库污染或资源占用做准备

六、经验教训

6.1 防御层面

  • 最小权限原则：私有Gitea默认应关闭注册，按需手动添加用户
  • 监控告警：用户数量异常增长应触发告警
  • 邮箱域名黑名单：维护已知垃圾域名列表

6.2 应急层面

  • 先备份再操作：批量删除前执行 gitea dump 备份
  • 预览再执行：先打印待删除列表确认，避免误伤
  • 容器用户权限：记住Gitea必须使用 git 用户运行管理命令

6.3 攻击模式识别

此类攻击属于低慢速自动化注册攻击：

  • 不暴力破解，模拟真实用户行为
  • 使用算法生成域名和用户名，规避简单规则检测
  • 批量创建账户后长期潜伏，为后续利用做准备
